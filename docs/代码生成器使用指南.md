# MilitaryTrain 代码生成器使用指南

## 概述

代码生成器根据数据库表结构，一键生成完整的前后端 CRUD 代码，包括：

- **后端 6 个文件**：Controller、Service、DAO、DO（SQLAlchemy 模型）、VO（Pydantic 模型）、菜单 SQL
- **前端 2 个文件**：API 调用模块（JS）、CRUD 页面（Vue）

生成后只需复制文件到工程对应目录、执行 SQL、重启服务即可使用。

---

## 一、整体流程

```
创建数据库表 → 导入到代码生成器 → 配置字段和生成信息 → 预览 → 生成 ZIP
    → 复制文件到工程 → 执行菜单 SQL → 创建字典 → 重启服务 → 完成
```

---

## 二、模板类型选择

代码生成器支持三种模板类型，在"生成信息"标签页的"生成模板"中选择：

### 2.1 crud（单表增删改查）

适用于普通的平面表格，有搜索栏、分页、新增/编辑弹窗。

**适用于**：用户管理、连队管理、成绩项目、签到任务等。

### 2.2 tree（树形结构）

适用于有层级关系的数据，页面上是可展开/折叠的树形表格。

**前提条件**：表必须包含三个字段：
- 一个 ID 字段（如 `dept_id`）
- 一个父 ID 字段（如 `parent_id`）
- 一个名称字段（如 `dept_name`）

**适用于**：部门管理、菜单管理、分类管理等。

选择 tree 模板后，需要在"生成信息"中额外配置三个字段：
- **树编码字段**：选择 ID 列（如 `dept_id`）
- **树父编码字段**：选择父 ID 列（如 `parent_id`）
- **树名称字段**：选择名称列（如 `dept_name`）

### 2.3 sub（主子表）

适用于一对多关系，编辑主表时弹窗中会内嵌子表的编辑表格。

**前提条件**：
- 主表和子表都已导入到代码生成器
- 子表中有外键列关联主表

**适用于**：考试+考题、订单+订单明细等。

选择 sub 模板后，需要在"生成信息"中额外配置：
- **关联子表的表名**：下拉选择子表
- **子表关联的外键名**：选择子表中指向主表的字段

## 三、生成信息各字段说明

在代码生成页点击"编辑"→"生成信息"标签页，需要填写以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| **生成模板** | crud / tree / sub，决定页面交互模式 | `crud` |
| **前端类型** | **必须选"Vue3 Element Plus"**（当前项目用 Vue3） | `element-plus` |
| **生成包路径** | Python 包路径，决定后端代码目录 | `module_military` |
| **生成模块名** | 模块名，决定前端目录和权限前缀 | `military` |
| **生成业务名** | 英文业务名，决定类名、文件名、URL | `company` |
| **生成功能名** | 中文描述，出现在页面标题和注释中 | `连队管理` |
| **生成代码方式** | 选"zip压缩包"即可 | `0` |
| **上级菜单** | 新菜单挂在哪个现有菜单下 | 选"军训管理" |

### 各字段对生成代码的影响

假设配置为：
- packageName = `module_military`
- moduleName = `military`
- businessName = `company`
- functionName = `连队管理`

生成的代码结构和命名：

```
后端文件：
  module_military/controller/company_controller.py    → URL: /military/company/list
  module_military/service/company_service.py
  module_military/dao/company_dao.py
  module_military/entity/do/company_do.py              → 表模型 Company
  module_military/entity/vo/company_vo.py              → Pydantic 模型

前端文件：
  src/api/military/company.js                          → API 调用
  src/views/military/company/index.vue                 → CRUD 页面

SQL：
  backend/sql/company_menu.sql                         → 菜单+权限

权限标识符：
  military:company:list     (查询)
  military:company:add      (新增)
  military:company:edit     (修改)
  military:company:remove   (删除)
  military:company:export   (导出)
```

## 四、字段信息配置详解

导入表后，系统会自动推断每列的配置。进入"编辑"→"字段信息"标签页逐列核对调整：

### 4.1 各配置项含义

| 配置项 | 说明 | 可选值 |
|--------|------|--------|
| **Python类型** | 映射到的 Python 类型 | str, int, float, Decimal, datetime, bool |
| **Java字段名** | 驼峰命名的字段名 | 自动从列名转换，可手动改 |
| **插入** | 新增时是否显示此字段 | 勾选/不勾选 |
| **编辑** | 编辑时是否显示此字段 | 勾选/不勾选 |
| **列表** | 表格中是否显示此列 | 勾选/不勾选 |
| **查询** | 搜索栏是否有此条件 | 勾选/不勾选 |
| **查询方式** | 搜索时的匹配方式 | =, !=, >, >=, <, <=, Like, BETWEEN |
| **必填** | 新增/编辑时是否必填校验 | 勾选/不勾选 |
| **唯一** | 值是否必须唯一 | 勾选/不勾选 |
| **显示类型** | 前端表单控件类型 | 见下方 |
| **字典类型** | 关联的字典（下拉/单选自动加载选项） | 如 `sys_normal_disable` |

### 4.2 显示类型说明

| 显示类型 | 前端渲染 | 适用场景 |
|----------|----------|----------|
| `input` | 文本输入框 | 名称、编号等短文本 |
| `textarea` | 多行文本框 | 备注、描述等长文本 |
| `select` | 下拉选择框 | 类型、状态（配合字典类型使用） |
| `radio` | 单选框 | 性别、状态等有限选项 |
| `checkbox` | 多选框 | 多选场景 |
| `datetime` | 日期时间选择器 | 时间字段 |
| `imageUpload` | 图片上传 | 头像、照片 |
| `fileUpload` | 文件上传 | 附件 |
| `editor` | 富文本编辑器 | 内容、详情 |

### 4.3 常用配置惯例

| 列名特征 | 推荐配置 |
|----------|----------|
| `*_id`（主键） | 不勾选"插入"和"编辑"，勾选"列表" |
| `*_name`, `*_code` | 查询方式设为 **Like**（模糊搜索） |
| `*_time`, `*_date` | 显示类型设为 **datetime**，查询方式设为 **BETWEEN**（范围查询） |
| `status` | 显示类型设为 **radio**，字典类型选 **sys_normal_disable** |
| `*_type`, `*_sex` | 显示类型设为 **select**，配置对应字典 |
| `remark` | 不勾选"查询"，勾选"列表"即可 |
| `create_by`, `create_time` | 不勾选"插入"（自动赋值），勾选"列表" |
| `update_by`, `update_time` | 不勾选"插入"和"编辑"（自动赋值），勾选"列表" |
| `*_image`, `*_photo`, `*_avatar` | 显示类型设为 **imageUpload** |
| `*_content` | 显示类型设为 **editor** |
| 长度 ≥ 500 的字符串列 | 显示类型自动设为 **textarea** |

## 五、操作步骤速查

### 步骤 1：创建数据库表

```sql
-- 示例：创建连队表
CREATE TABLE sys_military_company (
    company_id    BIGSERIAL PRIMARY KEY,
    company_name  VARCHAR(50)  NOT NULL,
    company_code  VARCHAR(20)  NOT NULL,
    status        CHAR(1) DEFAULT '0',
    create_by     VARCHAR(64) DEFAULT '',
    create_time   TIMESTAMP,
    update_by     VARCHAR(64) DEFAULT '',
    update_time   TIMESTAMP,
    remark        VARCHAR(500)
);
COMMENT ON TABLE sys_military_company IS '军训连队表';
COMMENT ON COLUMN sys_military_company.company_name IS '连队名称';
COMMENT ON COLUMN sys_military_company.company_code IS '连队编号';
COMMENT ON COLUMN sys_military_company.status IS '状态（0正常 1停用）';
```

> **重要**：列注释（COMMENT）必须写清楚，代码生成器会用注释内容作为页面上的字段标签。如果不写注释，页面上的字段标签会显示列名而不是中文名。

### 步骤 2：导入表

1. 打开 **系统工具 → 代码生成**
2. 点击 **导入** 按钮
3. 在弹出的对话框中勾选目标表（可多选）
4. 点击确定

系统会自动分析表结构，为每列推断合适的 Python 类型、显示类型和查询方式。

### 步骤 3：编辑配置

点击表行右侧的 **编辑** 按钮，进入配置页面。

**基本信息标签页**：
- 核对表名称和描述
- 设置实体类名称（PascalCase，如 `Company`）
- 设置作者

**字段信息标签页**：
- 逐列核对自动推断的配置
- 重点调整：显示类型、查询方式、必填/唯一、字典类型

**生成信息标签页**：
- 选择生成模板（crud/tree/sub）
- 前端类型选 **Vue3 Element Plus**
- 填写包路径、模块名、业务名、功能名
- 选择上级菜单
- 生成方式选 zip 压缩包

### 步骤 4：预览

点击表行右侧的 **预览** 按钮，逐个检查生成的代码内容，确认无误后关闭。

### 步骤 5：生成下载

1. 勾选表前面的复选框
2. 点击 **生成** 按钮
3. 浏览器自动下载 ZIP 文件

### 步骤 6：集成到工程

```bash
# 解压 ZIP
unzip ruoyi.zip -d gen_output

# 复制后端文件
cp gen_output/backend/<package_path>/controller/*_controller.py \
   military-train-backend/<对应模块>/controller/
cp gen_output/backend/<package_path>/service/*_service.py \
   military-train-backend/<对应模块>/service/
cp gen_output/backend/<package_path>/dao/*_dao.py \
   military-train-backend/<对应模块>/dao/
cp gen_output/backend/<package_path>/entity/do/*_do.py \
   military-train-backend/<对应模块>/entity/do/
cp gen_output/backend/<package_path>/entity/vo/*_vo.py \
   military-train-backend/<对应模块>/entity/vo/

# 复制前端文件
cp gen_output/frontend/api/<module>/*.js \
   military-train-frontend/src/api/<module>/
mkdir -p military-train-frontend/src/views/<module>/<business>
cp gen_output/frontend/views/<module>/<business>/index.vue \
   military-train-frontend/src/views/<module>/<business>/

# 执行菜单 SQL
docker exec -i <pg容器名> psql -U postgres -d military-train \
   < gen_output/backend/sql/*_menu.sql
```

### 步骤 7：创建字典（如需要）

如果字段信息中配置了字典类型，需要先在 **系统管理 → 字典管理** 中创建对应的字典和字典项。

### 步骤 8：重启服务

```bash
# 后端（路由自动注册、模型自动建表）
pkill -f "python app.py"
cd military-train-backend
nohup .venv/bin/python app.py > logs/server.log 2>&1 &

# 前端不需要重启（Vite 热更新自动检测）
```

### 步骤 9：分配权限

用管理员账号登录，在 **系统管理 → 角色管理** 中给对应角色分配新菜单的权限，然后相关用户重新登录即可看到新菜单。

## 六、数据库表设计规范

为了让代码生成器产出高质量的代码，建表时遵循以下规范：

1. **主键命名**：`<表简称>_id`，如 `company_id`、`student_id`
2. **通用字段**：每张表都应有 `create_by`、`create_time`、`update_by`、`update_time`、`remark`（系统会自动处理不显示在新增/编辑表单中）
3. **状态字段**：命名为 `status`，类型 `CHAR(1)`，值用 `'0'` 和 `'1'`
4. **列注释**：必须用 COMMENT 写清楚，这会成为页面上字段的标签文本
5. **外键关系**：子表的外键列名与被引用的主表主键名一致
6. **表名前缀**：统一用 `sys_military_` 或其他前缀，标识模块归属

## 七、常见问题

### Q: 生成完代码后路由不生效？

A: 后端路由是自动发现的，确保 controller 文件放在了 `*/controller/` 目录下，并且文件中包含 `APIRouterPro` 实例。重启后端即可自动注册。

### Q: 前端新菜单不显示？

A: 检查：1) 菜单 SQL 是否执行成功；2) 角色是否分配了新菜单的权限；3) 用户是否重新登录。

### Q: 生成的表不存在？

A: 检查 DO 文件是否正确放在了 `entity/do/` 目录下。服务启动时会自动调用 `init_create_table()` 检测并创建新的数据库表。

### Q: 下拉框没有选项数据？

A: 需要先在"字典管理"中创建对应的字典类型和字典数据项。下拉框/单选框的数据来自字典。

### Q: 生成后改了表结构怎么同步？

A: 在代码生成页面点击表行的 **同步** 按钮，系统会自动对比数据库和配置的差异，新增/删除列，同时保留已有的自定义配置。同步后重新生成代码覆盖。

### Q: 能不能直接写入工程而不是下载 ZIP？

A: 可以，在生成信息中把"生成代码方式"选为"自定义路径"。但需要先将 `config/env.py` 中的 `GenSettings.allow_overwrite` 设为 `True`。建议用 ZIP 方式更安全可控。

## 八、代码生成器不能做的事

以下功能需要手工编写：

- 复杂的业务逻辑（如审批流程、签到判断、自动计算）
- 数据可视化大屏（ECharts 图表）
- 非标准的前端交互（如扫码签到、拖拽排序）
- 跨表关联查询和统计报表
- 自定义的导入导出逻辑
- 消息推送和通知
- 定时任务逻辑

代码生成器解决的是 **标准 CRUD 的重复劳动**，让你把精力集中在业务逻辑上。
